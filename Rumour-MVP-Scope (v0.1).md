Rumour: MVP Scope (v1.0)

1. Executive Summary

Rumour is a hyper-local, ephemeral discovery tool designed to combat digital isolation by encouraging real-world community interaction. Unlike traditional social media, Rumour prioritizes presence over profiles and moments over feeds.

Core Concept: "The Buzz" – Real-time, location-based events that disappear as soon as they expire.

Mission: To bridge the gap between digital discovery and physical community.

2. Product Philosophy: "The Anti-Scroll"

To ensure Rumour doesn't become another "time-sink," the following principles guide the MVP:

Low Dwell Time: Users find what's happening and put the phone away immediately.

High Intent: Features are built solely to drive users to a physical location.

Ephemeral by Default: Content has a strict expiration date (default 4 hours).

Proximity-Gated: Detailed information is unlocked only by physical presence (GPS).

The Universal Local: An "Explorer Mode" that adapts the UI to the current city.

3. MoSCoW Prioritization

3.1 Must-Have (Critical for Launch)

Pulse Map: Real-time heat clustering of active "Buzzes."

Timed Posts: Event creation with 4-hour auto-expiry.

Geofence Reveal: Blur/unblur logic based on 100m proximity.

Identity Verification: Phone or Stripe ID verification for human-only users.

Safety Reporting: 3-flag removal protocol.

3.2 Should-Have

Explorer Mode: Manual toggle for "City Skins" and UI adaptation.

AI Moderation: Real-time automated text scanning.

Smart Translation: Localized event descriptions for travelers.

3.3 Could-Have (Future)

Digital Tip Jar: Payment integration for performers.

The Vouch: Attendance-based reputation scores.

Anchor Dashboard: Business analytics for verified foot traffic.

4. User Stories

The Seeker: Wants a map that "glows" where the activity is to find local energy.

The Explorer: Wants the UI to change (e.g., neon theme in Tokyo) while filtering for local-only events.

The Performer: Wants a digital tip jar to monetize spontaneous physical crowds.

The Connection: Wants to find nearby "Third Places" (e.g., board game nights) for IRL interaction.
